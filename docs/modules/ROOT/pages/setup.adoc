= Setup

== Prerequisites

Before installing SysBot, ensure you have the following:

* **Python**: Version 3.7 or higher (Python 3.8+ recommended)
* **pip**: Python package installer
* **Git**: For cloning the repository (optional, for development)

=== Platform-Specific Requirements

==== Linux

[source,bash]
----
# Debian/Ubuntu
sudo apt-get update
sudo apt-get install python3 python3-pip

# RHEL/CentOS/Fedora
sudo dnf install python3 python3-pip

# Verify installation
python3 --version
pip3 --version
----

==== Windows

. Download Python from https://www.python.org/downloads/
. Run the installer and ensure "Add Python to PATH" is checked
. Open Command Prompt and verify:

[source,cmd]
----
python --version
pip --version
----

==== macOS

[source,bash]
----
# Using Homebrew
brew install python3

# Verify installation
python3 --version
pip3 --version
----

== Installation

=== From PyPI (Recommended)

The simplest way to install SysBot is using pip:

[source,bash]
----
pip install sysbot
----

To upgrade to the latest version:

[source,bash]
----
pip install --upgrade sysbot
----

=== From Source

For development or to get the latest unreleased features:

[source,bash]
----
# Clone the repository
git clone https://github.com/JoReci2/sysbot.git
cd sysbot

# Create a virtual environment (recommended)
python -m venv venv
source venv/bin/activate  # On macOS/Linux
# or
venv\Scripts\activate     # On Windows

# Install in development mode
pip install -e .
----

=== Using Virtual Environment (Recommended)

Using a virtual environment helps isolate project dependencies:

[source,bash]
----
# Create virtual environment
python -m venv sysbot-env

# Activate on Linux/macOS
source sysbot-env/bin/activate

# Activate on Windows
sysbot-env\Scripts\activate

# Install SysBot
pip install sysbot

# Deactivate when done
deactivate
----

== Dependencies

SysBot automatically installs the following dependencies:

* **robotframework**: Robot Framework automation framework
* **paramiko**: SSH implementation
* **sshtunnel**: SSH tunneling support
* **netmiko**: Multi-vendor library for SSH connections
* **redfish**: Redfish API support
* **pyVmomi**: VMware vSphere API Python bindings
* **pywinrm**: Python WinRM client
* **pyOpenSSL**: OpenSSL wrapper
* **pytz**: Timezone handling

== Verification

After installation, verify SysBot is correctly installed:

[source,bash]
----
# Check installation
pip show sysbot

# Try importing in Python
python -c "from sysbot.Sysbot import Sysbot; print('SysBot imported successfully')"
----

== Configuration

=== Basic Configuration

SysBot works out of the box with default settings. No additional configuration is required for basic usage.

=== Environment Variables

You can optionally set environment variables for common settings:

[source,bash]
----
# Example environment variables (optional)
export SYSBOT_DEFAULT_TIMEOUT=30
export SYSBOT_LOG_LEVEL=INFO
----

=== SSH Configuration

For SSH connections, you may want to configure your SSH client settings:

[source,bash]
----
# ~/.ssh/config example
Host *
    ServerAliveInterval 60
    ServerAliveCountMax 3
    StrictHostKeyChecking no
    UserKnownHostsFile=/dev/null
----

NOTE: Disabling StrictHostKeyChecking should only be done in test environments.

=== WinRM Configuration

For Windows Remote Management, ensure WinRM is enabled on target systems:

[source,powershell]
----
# On Windows target (run as Administrator)
winrm quickconfig -q
winrm set winrm/config/service/auth '@{Basic="true"}'
winrm set winrm/config/service '@{AllowUnencrypted="true"}'
----

WARNING: Allowing unencrypted WinRM should only be done in secure test environments.

== Troubleshooting

=== Common Installation Issues

==== pip Command Not Found

If `pip` is not found, try:

[source,bash]
----
python -m pip install sysbot
----

==== Permission Denied

On Linux/macOS, if you encounter permission errors:

[source,bash]
----
# Install for current user only
pip install --user sysbot
----

==== SSL Certificate Errors

If you encounter SSL certificate errors:

[source,bash]
----
# Use trusted host (not recommended for production)
pip install --trusted-host pypi.org --trusted-host files.pythonhosted.org sysbot
----

==== Dependency Conflicts

If you have dependency conflicts:

[source,bash]
----
# Create a fresh virtual environment
python -m venv fresh-env
source fresh-env/bin/activate
pip install sysbot
----

=== Network Requirements

SysBot requires network access to:

* Target systems you want to connect to
* PyPI (pypi.org) for installation

Ensure your firewall and proxy settings allow these connections.

=== Port Requirements

Default ports used by SysBot:

* **SSH**: 22 (TCP)
* **WinRM**: 5986 (TCP, HTTPS) or 5985 (TCP, HTTP)
* **HTTP**: 80 (TCP)
* **HTTPS**: 443 (TCP)

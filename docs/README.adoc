= SysBot Documentation

This directory contains the SysBot documentation in Antora AsciiDoc format.

== Structure

The documentation follows the https://docs.antora.org/[Antora] standard structure:

----
docs/
├── antora.yml              # Component descriptor
└── modules/
    ├── ROOT/               # English documentation (default)
    │   ├── nav.adoc       # Navigation menu
    │   └── pages/         # Documentation pages
    │       ├── index.adoc
    │       ├── setup.adoc
    │       ├── quickstart.adoc
    │       ├── robotframework.adoc
    │       ├── unittest.adoc
    │       ├── listener_plugin.adoc
    │       ├── developer.adoc
    │       └── references.adoc
    └── fr/                 # French documentation
        ├── nav.adoc       # Navigation menu (French)
        └── pages/         # Documentation pages (French)
            ├── index.adoc
            ├── setup.adoc
            ├── quickstart.adoc
            ├── robotframework.adoc
            ├── unittest.adoc
            ├── listener_plugin.adoc
            ├── developer.adoc
            └── references.adoc
----

== Building Documentation

=== Prerequisites

To build the documentation locally, you need:

* Node.js (for Antora)
* Antora CLI

Install Antora:

[source,bash]
----
npm install -g @antora/cli @antora/site-generator
----

=== Build Locally

From the repository root:

[source,bash]
----
# Build the documentation site
antora antora-playbook.yml

# The generated site will be in the public/ directory
----

=== View Documentation

After building, you can view the documentation:

[source,bash]
----
# Using Python's built-in HTTP server
cd public
python -m http.server 8000

# Then open http://localhost:8000 in your browser
----

== Contributing to Documentation

When contributing to the documentation:

1. Follow AsciiDoc syntax
2. Add new pages to `docs/modules/ROOT/pages/` or `docs/modules/fr/pages/`
3. Update the navigation file (`nav.adoc`) when adding new pages
4. Maintain consistency between English and French versions
5. Test your changes by building the documentation locally

=== Adding a New Page

. Create the page file in `docs/modules/ROOT/pages/newpage.adoc`
. Add the corresponding French translation in `docs/modules/fr/pages/newpage.adoc`
. Update both navigation files:
  * `docs/modules/ROOT/nav.adoc`
  * `docs/modules/fr/nav.adoc`
. Build and test the documentation
